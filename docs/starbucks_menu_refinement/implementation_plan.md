# スターバックスメニューの正規表現精査（フラペチーノ・ティーラテ）の実装計画

フラペチーノやティーラテの前に `.*` が使用されている箇所を、実際のレギュラーメニュー名に限定するように修正します。これにより、無関係な文字列との誤マッチングを防ぎます。

## ユーザーレビューが必要な事項

- **対象メニューの網羅性**: 以下のリストにないメニュー（季節限定など）はマッチしなくなりますが、よろしいでしょうか？
- **単体キーワードのマッチ**: 「フラペチーノ」や「ティー ラテ」という言葉単体でのマッチを許容するかどうか。（現在は許容する形に設計します）

## 変更内容

### starbucks_regex.py の修正

- `STARBUCKS_JP_DRINK_MENU_REGEX` 内の `.*?\s?ティー\s?ラテ` と `.*?\s?フラペチーノ` を修正します。
- 各単語間のスペースは `\s?` でオプションにします。
- **ティーラテのプレフィックス**: `抹茶|ほうじ茶|チャイ|アール\s?グレイ|カモミール|ゼン\s?クラウド\s?ウーロン|イングリッシュ\s?ブレックファスト|さくら`
- **フラペチーノのプレフィックス**: `コーヒー|エスプレッソ\s?アフォガート|キャラメル|ダーク\s?モカ\s?チップ(?:\s?クリーム)?|抹茶\s?クリーム|バニラ\s?クリーム|マンゴー\s?パッション\s?ティー|メロン\s?オブ\s?メロン|GOHOBI\s?メロン|ストロベリー|瀬戸内\s?レモン\s?ケーキ|さくら`
- ※レギュラーメニューに加え、一部の期間限定メニューも追加されました。

### tests/test_starbucks_regex.py の修正

- 具体的になったメニュー名でのテストケースを追加。
- 誤マッチング（例：「牛丼 ティー ラテ」など）が失敗することを確認するテストケースを追加。

## 検証計画

### 自動テスト
- `pytest` を実行し、既存のテストおよび追加したテストがすべて通ることを確認します。

### 手動確認
- 修正後の正規表現を `app.py` などで試し、意図した動作（広すぎるマッチングの解消）を確認します。
